Loomweave/MCP tools

MCP tools

loomweave serve exposes a 42+-tool consult-mode MCP surface that an agent calls instead of grep-and-read. Grouped below by family. Connect an MCP client to a live loomweave serve and read tools/list for the exact, version-matched schemas.

Note · default confidence is resolved
Graph-traversal tools (entity_callers_list, entity_neighborhood_get, entity_execution_path_list) return only resolved edges by default. Ambiguous static candidates and LLM-inferred edges are opt-in. Results carry a scope_excludes block naming static blind spots, so an empty section is never read as a guaranteed true negative. See Concepts → scope_excludes.

Navigation & graph traversal

Locate an entity, then walk the graph around it — callers, callees, one-hop neighborhood, bounded call paths, call sites, and subsystem membership.

ToolWhat it answers
entity_atWhich entity covers this file:line? Returns the innermost match plus a match_reason evidence block and index freshness.
entity_findFind entities by id, name, short name, summary, and docstring content (stemmed FTS + substring recall). Optional kind filter.
entity_callers_listWho calls this entity? resolved by default; carries scope_excludes.
entity_call_site_listThe concrete call sites — where, in source, this entity is invoked.
entity_execution_path_listBounded, calls-only execution paths from an entity, ranked longest-first, with truncation reporting.
entity_neighborhood_getThe one-hop neighborhood: callers, callees, container, contained, references, imports (imports_in/imports_out), and kind-tagged relations (relations_in/relations_out) in one call.
entity_relation_listWhat subclasses X / what does a decorator decorate / what implements a trait — the inherits_from/decorates/implements/derives edges, direction-explicit, each with the anchoring source line.
subsystem_member_listWhich module entities belong to this subsystem?
entity_subsystem_getThe reverse lookup — which subsystem does this entity belong to?

Briefing & source inspection

Get a one-paragraph explanation (lazy LLM, cached), the indexed source span, a one-shot orientation packet, and project-level status. summary is the only path that makes a network call to a model provider.

ToolWhat it answers
entity_summary_get (summary)A cached one-paragraph summary for one entity, dispatching the LLM lazily. Degrades to a deterministic structural-fallback if the model returns non-JSON — and caches that, so a retry is a free hit.
entity_summary_preview_cost_getWhat would summarising this entity cost before you spend it?
entity_source_getThe indexed source span and surrounding context for an entity.
entity_orientation_pack_getOne packet ≈ find + at + source + neighborhood + issues + freshness — the fastest way for an agent to orient on a single entity.
project_status_getProject-level health: entity counts, index freshness, finding totals.

Guidance, findings & federation context

Operator-authored guidance sheets, scanner findings, and the enrich-only sibling reach into Filigree (issues) and Wardline (taint facts). Every federation binding degrades cleanly when the sibling is absent.

ToolWhat it answers
entity_guidance_listWhich guidance sheets apply to this entity?
propose_guidance / promote_guidanceAuthor and promote guidance through the MCP path (operators also use loomweave guidance).
entity_finding_listScanner / imported findings anchored to this entity.
project_finding_listEvery finding across the project (no entity id) — each carrying its anchoring entity SEI + file:line + severity/rule, paginated.
entity_issue_list (issues_for)Filigree issues attached to this entity, drift-aware. Enrich-only: returns an unavailable envelope if Filigree is down rather than failing.
entity_wardline_get / entity_wardline_listWardline taint facts reconciled to the entity by qualname (Loomweave hosts the taint-fact store). Enrich-only.

Analyze lifecycle & freshness

Drive and observe the indexing pass, and diff the index against a prior run to see what moved.

ToolWhat it answers
analyze_startKick off an indexing run over MCP.
analyze_status_getProgress and terminal status of the current / last run.
analyze_cancelCancel an in-flight run.
index_diff_getWhat changed between this index and a prior one — added, removed, moved entities.

Faceted & shortcut queries

Targeted structural queries that eliminate exploration — entry points, dead code, circular imports, coupling hotspots, churn, tests, routes, data models, deprecations, todos, and semantic search.

ToolWhat it answers
entity_kind_list / entity_tag_listBrowse entities by kind or by tag (the way to locate a subsystem without visually filtering).
entity_entry_point_listWhere does execution start? Entry points into the graph.
entity_dead_listReachability shortcut — entities with no resolved callers (honest about unresolved call sites; fails toward live).
module_circular_import_listImport cycles between modules.
entity_coupling_hotspot_listThe most-coupled entities — where change ripples.
entity_high_churn_list / entity_recent_change_listWhat changes a lot, and what changed recently.
entity_test_list / entity_test_caller_listThe tests, and what tests a given entity.
entity_http_route_list / entity_data_model_listHTTP routes and data models surfaced as facets.
entity_deprecation_list / entity_todo_listDeprecations and in-source TODOs.
entity_semantic_search_listEmbedding-ranked semantic search — opt-in, when embeddings are available (distinct from the always-on entity_find).

This page presents a curated, family-grouped view of the surface. The live server is the authority for the complete, current tools/list and the exact parameter schemas — and the surface grows. See also the CLI reference.
