## Python Language Plugin (`clarion-plugin-python`)

**Location:** `plugins/python/`

**Responsibility:** Owns Python-source ingestion for Clarion — parses `*.py` files with the CPython `ast` stdlib, emits `module`/`class`/`function` entities with 3-segment stable EntityIds, anchors `contains` and `imports` edges directly, and delegates type-resolved `calls`/`references` edges to a managed pyright-langserver subprocess. The only Loom-suite surface that probes Wardline (`wardline_probe.py`) for the L8 integration handshake. Public surface to the Rust core is the five JSON-RPC methods declared in `server.py` plus the manifest at `plugin.toml`.

**Key Components:**
- `__main__.py` (15 LOC) — entry point bound to `[project.scripts] clarion-plugin-python = ...:main`; installs stdio guard then calls `serve()`. `server.py:main()` performs `install_stdio()` internally; `__main__` is a thin `sys.exit` wrapper (`__main__.py:14-15`).
- `server.py` (300 LOC) — Content-Length framing reader/writer (`read_frame`/`write_frame`, lines 74-129), dispatch table and five method handlers (`dispatch`, lines 247-276), and the per-25-files pyright restart policy (`MAX_FILES_PER_PYRIGHT_SESSION = 25` at `server.py:49`; recycle logic `server.py:219-223`). Frame cap: `MAX_CONTENT_LENGTH = 8 MiB` (`server.py:48`). `PyrightRunState` is passed through every new `PyrightSession` so the 3-restart budget is run-wide across recycles (`server.py:71`, `server.py:201`).
- `extractor.py` (1052 LOC) — top-level `extract_with_stats` (lines 332-399); three AST walks: `_walk` recursion emitting entities + `contains` edges, `_ImportEdgeCollector` (NodeVisitor), `_ReferenceSiteCollector` (NodeVisitor). **New since prior catalog:** `DefinitionSpan` TypedDict (line 100) with `decl_line`/`body_line_start`/`decorator_line_*` fields, stamped into every function/class entity at `extractor.py:1009`; `FunctionSignature`/`ClassSignature` TypedDicts (lines 126-146) and `SIGNATURE_SCHEMA_VERSION = 1` (line 123), implementing ADR-038 REQ-C-01; `_definition_span` helper (lines 903-930) expands entity `source_range.start_line` up to the topmost decorator for `entity_at` resolution; `_function_signature`/`_class_signature` builders (lines 958-979). Module entities explicitly omit `signature` (line 278 note).
- `pyright_session.py` (1427 LOC) — `PyrightSession` subprocess lifecycle, LSP framing, `_FunctionIndex` builder, `resolve_calls` (callHierarchy) and `resolve_references` (definition/typeDefinition). **New since prior catalog:** `PyrightRunState` dataclass (lines 44-57) shared across session recycles to prevent the 3-restart cap from resetting at every 25-file boundary; `PYRIGHT_EXCLUDE_PATTERNS` now includes `.jj/` (`pyright_session.py:67-76`). Constants verified: `MAX_PYRIGHT_RESTARTS_PER_RUN = 3` (line 61), `MAX_REFERENCE_SITES_PER_FILE = 2000` (line 62), `PYRIGHT_INIT_TIMEOUT_SECS = 30.0` (line 63).
- `entity_id.py` (75 LOC) — L2 3-segment EntityId assembler; mirrors Rust validator (`[a-z][a-z0-9_]*` grammar, no `:`, no empty segment). `entity_id.py:23-75`.
- `qualname.py` (48 LOC) — reconstructs CPython `__qualname__` from AST parent chain (`qualname.py:34-48`).
- `reference_resolver.py` (70 LOC) — `ReferenceResolver` protocol + `NoOpReferenceResolver`; `ReferenceSite`/`ReferencesEdgeProperties` TypedDicts.
- `call_resolver.py` (65 LOC) — `CallResolver` protocol + `NoOpCallResolver`; `CallsRawEdge`/`Finding`/`UnresolvedCallSite` TypedDicts.
- `wardline_probe.py` (56 LOC) — `probe()` at lines 35-56 uses `importlib.import_module("wardline.core.registry")` (line 38) to test for Wardline presence before separately importing `wardline` for `__version__`. Returns `absent`/`enabled`/`version_out_of_range`. The `wardline.core.registry` direct import is the **federation asterisk** (loom.md §5) — still present, not yet migrated to NG-25 descriptor read. Tracked as open Filigree issue `clarion-1f6241b329`.
- `stdout_guard.py` (62 LOC) — `_GuardedTextStdout` raises `StdoutGuardError` on any write (`stdout_guard.py:36-41`); `install_stdio()` (lines 57-62) captures real stdin/stdout buffers before installing the guard.

**Dependencies:**
- Inbound: Rust plugin host (`crates/clarion-core/` plugin host, `crates/clarion-mcp/`). Spawns the binary, speaks Content-Length-framed JSON-RPC, consumes `entities`/`edges`/`stats`. No Python-layer inbound consumers.
- Outbound (process-level only, no Rust crate deps): `pyright-langserver --stdio` subprocess (`pyright_session.py:~637`), pinned to `1.1.409` (`pyproject.toml:20`, `plugin.toml:29`). Runtime Python deps: `packaging>=24` for Wardline version parsing (`wardline_probe.py:30`).
- Wardline coupling (federation asterisk): `importlib.import_module("wardline.core.registry")` at `wardline_probe.py:38` — fail-soft (ImportError → `{"status": "absent"}`), but the direct module path is a named initialization-coupling asterisk in `loom.md §5`. **Current status:** asterisk live; Wardline-side prerequisite (NG-25 descriptor, SP2) is described as met in `loom.md §5` but Clarion has not migrated; `clarion-1f6241b329` is open/ready at P2.
- Cross-language fixture: `fixtures/entity_id.json` consumed by `tests/test_entity_id.py:25` — byte-for-byte parity proof for L2 EntityId construction shared with Rust tests.

**Patterns Observed:**
- **Pyright-as-a-service with run-wide session recycling.** `PyrightSession` is recycled every `MAX_FILES_PER_PYRIGHT_SESSION = 25` files (`server.py:49`, recycle `server.py:219-223`) to bound memory growth. The 3-restart budget now shared via `PyrightRunState` across recycles (`pyright_session.py:44-57`, wired at `server.py:71`), fixing a prior gap where the cap would reset at each 25-file boundary.
- **3-walk AST extraction.** Three separate `ast.NodeVisitor` / recursive walks over the same tree: `_walk` for entities + `contains` edges, `_ImportEdgeCollector` for `imports` edges, `_ReferenceSiteCollector` for reference sites. A fourth walk lives in `pyright_session._build_function_index`. Each walk has slightly different scope bookkeeping (extractor.py lines 396-659; pyright_session.py ~893+).
- **Fail-soft degradation to CLA-PY-PYRIGHT-* findings.** Three-tier resilience: timeouts → restart cap → permanent `_disabled` flag (`pyright_session.py:61`, `FINDING_PYRIGHT_POISON_FRAME` at `pyright_session.py:35`). All failure modes return zero edges + a `Finding` rather than raising.
- **Cross-language entity-ID parity.** `entity_id.py` intentionally duplicates Rust validator logic rather than sharing it; `fixtures/entity_id.json` is the cross-language contract.
- **Stdout guard via type substitution.** `_GuardedTextStdout` (stdout_guard.py:25-41) raises on every write to protect the JSON-RPC channel from stray library output.
- **SEI signature emission (new, ADR-038).** `extractor.py` now emits a `signature` field on every function and class entity — `{ "v": 1, "params": [...], "return_ann": "..." }` / `{ "v": 1, "bases": [...] }` — per `_function_signature`/`_class_signature` (lines 958-979). Modules emit `null` (no `signature` key). `plugin.toml` declares `[signature] schema_version = 1` with per-kind field lists (plugin.toml lines 58-72).
- **Definition-span entity context (new, clarion-460def6a51).** `DefinitionSpan` (extractor.py:100) now stamped into every function/class entity's `definition` field. Decorator presence expands `source_range.start_line` to the topmost decorator line so `entity_at` queries on decorator lines resolve to the enclosing entity. Logic in `_definition_span` (lines 903-930).

**Concerns:**
- **[HIGH] system-design.md §2 Python-plugin specifics stale — describes a parser that does not exist.** `system-design.md:213` (last-updated 2026-05-19) says "Tree-sitter for structural extraction … LibCST for decorator argument extraction." The actual implementation uses the CPython `ast` stdlib exclusively — no tree-sitter or LibCST dependency exists anywhere in `plugins/python/` (not in `pyproject.toml`, not in any import). The entire "Parser dispatch," "Import resolution," "Decorator detection" paragraph block (lines 213-228) describes a design that was never implemented. This is the highest-severity drift item: a reader following the system-design to understand the parser will be misled.
- **[HIGH] Wardline federation asterisk still live despite Wardline-side prerequisite being met.** `wardline_probe.py:38` still calls `importlib.import_module("wardline.core.registry")`. `loom.md §5` records that the Wardline-side NG-25 descriptor shipped in SP2 and the `wardline.core.registry` surface no longer exists in the rebuild. The fail-soft means no runtime break today, but against a rebuilt Wardline the probe will always return `absent`. `clarion-1f6241b329` is open/ready (P2, no blockers). Fix: replace `importlib.import_module("wardline.core.registry")` with a descriptor file read per ADR-018 retirement path.
- **[MEDIUM] Prior-catalog error: `MAX_FILES_PER_PYRIGHT_SESSION` listed as `49` in description body.** The prior catalog's component listing (`catalog-python-plugin.md:12`) says `MAX_FILES_PER_PYRIGHT_SESSION = 49`; the confidence note corrects this to `25`. Current source at `server.py:49` confirms `25`. No code defect — the prior catalog had a documentation error.
- **[MEDIUM] `pyright_session.py` is 1427 LOC in a single module.** Grew by 21 lines since prior catalog (1406 → 1427). Still contains the `PyrightSession` class body plus ~600 lines of free-function helpers. The AST-index builder (`_build_function_index`, `_collect_entities`) duplicates traversal work already done by `extractor.py`. No new split has occurred; the concern from the prior catalog remains valid.
- **[MEDIUM] Four distinct AST walks over the same tree.** Three in `extractor.py`, one in `pyright_session._build_function_index`. Future changes touching scope semantics (e.g., `match`/`case` bindings, walrus operator) have four call sites to keep in sync.
- **[LOW] `system-design.md §2` mentions `TYPE_CHECKING` block exclusion, `alias_of` edges, and `python:unresolved:{}` placeholder entities — none of these exist in the implementation.** The import-resolution section (system-design.md:215-220) describes features deferred to v0.2; the code emits `imports` edges by AST-visiting `ast.Import`/`ast.ImportFrom` without virtualenv introspection, without `TYPE_CHECKING` exclusion, and without unresolved placeholder entities. Low severity because these are labelled as deferred features, not regressions.
- **[LOW] `StdoutGuardError` surfaces as `_ERR_INTERNAL` JSON-RPC response rather than a hard exit.** `dispatch`'s broad `except Exception` (server.py:274) catches any `StdoutGuardError` raised by a handler and converts it to a well-formed error response. The design implication — that a guard violation looks like a normal RPC error from the host's perspective — is not documented. No observable bug given the guard is there to prevent corruption, but the failure mode is subtle.
- **[LOW] Pyright opt-in marker means a "green" test run may have skipped LSP path.** `pyproject.toml:93-94` marks pyright-dependent tests with `@pytest.mark.pyright`; without pyright on PATH the full LSP code path is untested. Tests not labelled `slow` or `pyright` still exercise the noop resolvers thoroughly.
- **[INFO] Version bumped to 1.1.0.** `pyproject.toml:7`, `plugin.toml:4`, and `__init__.py:3` all show `1.1.0` (vs `1.0.0` at prior catalog date). Consistent across all three version sites.
